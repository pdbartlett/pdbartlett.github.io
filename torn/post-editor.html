Torn Forum Markdown Styler

Instructions & Quick Reference

  • Purpose: To make it as easy and quick as possible to produce nice looking forum posts, messages and newsletters in Torn.
  • Why Markdown? It's easy and more compact to write than HTML, but permits HTML instead if you prefer. Here is a cheat sheet to get you started.
  • Why CSS? To allow consistent styling without repeating yourself. Unfortunately Torn only allows style information to be use inside the HTML tags, so this tool applies the specifed CSS rules inline after converting the Markdown to HTML. Here is another cheat sheet, this time for CSS.
  • Post Container: All rendered Markdown is automatically wrapped in a .post-card container. Target .post-card in CSS to set outer margins, padding, and background color.
  • Inline Attribute Syntax: Append {#id}, {.class} or a mix of the two to the end of headings or paragraphs to target them specifically in your CSS (e.g., # Section Title {.alert}).
  • HTML Blocks: Keep inline HTML tags (like <div class="header-line">text</div>) closed on a single line so they don't interfere with Markdown parsing.
  • Theme Compatibility: Use the Switch Preview Theme button to check legibility against both dark and light Torn themes. Ensure text colors pair with explicit element backgrounds to force the issue, at the expense of ignoring user preferences, or use midtones which work well against either dark or light backgrounds.
